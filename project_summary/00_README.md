## 🔬 技术栈

- **基座模型**：Qwen2.5-0.5B-Instruct
- **嵌入模型**：Qwen3-Embedding-0.6B
- **离散化**：FAISS Residual Quantizer（3 层，codebook=256）
- **训练框架**：Transformers + Accelerate
- **评估**：约束解码 + HR@K / NDCG@K
- **数据来源**：Amazon Reviews 2023（Cell Phones & Accessories）

## 📝 核心结论

1. **生成式推荐在小候选集（N ≤ 20K）下可行**，HR@10 可达 3-4%
2. **候选集增大导致性能显著衰减**，5K → 100K 下降约 40%
3. **Beam=50 是最优选择**，Beam=100 收益极小
4. **大候选集场景建议混合架构**：召回截断 + 生成式推荐
5. **模型容量是瓶颈**：0.5B 参数难以记忆 100K 商品的 SID 映射
