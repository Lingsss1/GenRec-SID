# 模型训练详解（Phase 5）

## 一、基座模型

| 属性 | 值 |
|------|-----|
| 模型 | Qwen/Qwen2.5-0.5B-Instruct |
| 架构 | Qwen2ForCausalLM |
| 层数 | 24 |
| Hidden Size | 896 |
| 原始词表 | ~151K |

### 词表扩展

添加 **768 个 SID token**（256 × 3 层 RQ-VAE 码本）：
- 第 1 层：`<a_0>`, `<a_1>`, ..., `<a_255>`
- 第 2 层：`<b_0>`, `<b_1>`, ..., `<b_255>`
- 第 3 层：`<c_0>`, `<c_1>`, ..., `<c_255>`

扩展后总词表约 **152K**。

## 二、多任务 SFT 训练

采用 **ConcatDataset** 将三个任务的数据集拼接，统一进行 SFT 训练。

### 2.1 主任务：SidSFTDataset

**目标**：学习从历史交互 SID 序列预测下一个商品的 SID。

**Prompt 格式**：
```
### Instruction:
Based on the user's historical interactions, predict the next item's SID.

### Input:
History: <a_34><b_215><c_140>, <a_11><b_88><c_201>, <a_5><b_42><c_99>

### Response:
<a_77><b_156><c_33>
```

### 2.2 辅助任务 1：SidItemFeatDataset

**目标**：建立 SID 与商品标题之间的双向映射。

**双向 Prompt**：

方向 1（SID → Title）：
```
### Instruction:
Given the item's SID, predict its title.

### Input:
SID: <a_77><b_156><c_33>

### Response:
Apple iPhone 14 Pro Max Case
```

方向 2（Title → SID）：
```
### Instruction:
Given the item's title, predict its SID.

### Input:
Title: Apple iPhone 14 Pro Max Case

### Response:
<a_77><b_156><c_33>
```

### 2.3 辅助任务 2：FusionSeqRecDataset

**目标**：融合序列推荐与语义理解，从历史 SID 预测目标商品标题。

**Prompt 格式**：
```
### Instruction:
Based on the user's historical interactions, predict the next item's title.

### Input:
History: <a_34><b_215><c_140>, <a_11><b_88><c_201>

### Response:
Samsung Galaxy S24 Screen Protector
```

### 2.4 多任务设计的意义

| 任务 | 作用 |
|------|------|
| 主任务 | 核心推荐能力：序列 → SID |
| 辅助 1 | SID-语义对齐：让模型理解 SID 的语义含义 |
| 辅助 2 | 跨模态融合：增强模型对商品语义的理解 |

三个任务协同训练，使模型不仅学会 SID 之间的序列模式，还理解 SID 背后的语义信息。

## 五、训练流程

```
1. 加载基座模型 Qwen2.5-0.5B-Instruct
2. 从 index.json 提取 SID token，扩展 tokenizer 词表
3. resize_token_embeddings 扩展模型 embedding 层
4. 构建三任务 ConcatDataset
5. 配置 Trainer（bf16, gradient_checkpointing, early_stopping）
6. 训练，每 epoch 在 valid 集上评估 loss
7. 保存 final_checkpoint
8. 记录训练日志到 train_log.csv
9. （可选）上传日志到 wandb
```

## 八、GRPO 强化学习训练（Phase 6）

### 8.1 算法：Group Relative Policy Optimization (GRPO)

基于 MiniOneRec 项目的 RL 方案，在 SFT 模型基础上进行 GRPO 强化学习训练，进一步优化推荐性能。

**核心思想**：对同一个 prompt 生成一组（group）候选 SID，通过组内相对奖励计算优势（advantage），鼓励高 reward 的生成、抑制低 reward 的生成。

```
prompt → 生成 G 个候选 SID → 计算各自 reward → 组内归一化得到 advantage → 策略梯度更新
```

### 8.2 训练配置

| 参数 | 值 | 说明 |
|------|-----|------|
| 基座模型 | M_N_5K SFT checkpoint | SFT 训练后的模型 |
| num_generations | 4 | 每个 prompt 生成 4 个候选 |
| reward_type | rule | 完全匹配奖励（命中=1，未命中=0） |
| beta | 0.04 | KL 散度惩罚系数 |
| learning_rate | 1e-6 | 学习率 |
| train_batch_size | 4 | 单卡批大小 |
| gradient_accumulation | 4 | 梯度累积步数 |
| num_train_epochs | 1 | 训练轮数 |
| max_completion_length | 10 | 最大生成长度（SID 约 5 token） |
| temperature | 1.0 | 采样温度 |
| sample | 1000 | 主任务训练样本数 |
| seq_title_sample | 500 | 辅助任务训练样本数 |
| GPU | 2 x RTX 3090 | 主模型 GPU 0，ref_model GPU 1 |

### 8.3 多任务 RL 训练

与 SFT 阶段类似，RL 训练也采用多任务方式：

| 数据集 | 任务 | 样本数 |
|--------|------|--------|
| RLSidDataset | 历史 SID → 目标 SID | 1000 |
| RLTitle2SidDataset | 标题 → SID | 1000 |
| RLSeqTitle2SidDataset | 标题序列 → SID | 500 |

### 8.4 关键实现细节

1. **约束解码**：使用 `prefix_allowed_tokens_fn`（transformers 原生 API）确保生成的 SID 合法，基于前缀哈希表限制每步可选 token
2. **Reference Model**：独立的 ref_model 放在第二张 GPU 上，用于计算 KL 散度惩罚，防止策略偏离初始模型太远
3. **奖励函数**：rule reward — 生成的 SID 与 ground truth 完全匹配得 1.0 分，否则 0.0 分

### 8.5 训练输出

- RL 模型保存在：`train/runs/RL_M_N_5K/final_checkpoint/`
- 本地备份：`models/RL_M_N_5K_final_checkpoint/`
