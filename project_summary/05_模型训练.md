# 模型训练详解（Phase 5）

## 一、基座模型

| 属性 | 值 |
|------|-----|
| 模型 | Qwen/Qwen2.5-0.5B-Instruct |
| 架构 | Qwen2ForCausalLM |
| 层数 | 24 |
| Hidden Size | 896 |
| 原始词表 | ~151K |

### 词表扩展

添加 **768 个 SID token**（256 × 3 层 RQ-VAE 码本）：
- 第 1 层：`<a_0>`, `<a_1>`, ..., `<a_255>`
- 第 2 层：`<b_0>`, `<b_1>`, ..., `<b_255>`
- 第 3 层：`<c_0>`, `<c_1>`, ..., `<c_255>`

扩展后总词表约 **152K**。

## 二、多任务 SFT 训练

采用 **ConcatDataset** 将三个任务的数据集拼接，统一进行 SFT 训练。

### 2.1 主任务：SidSFTDataset

**目标**：学习从历史交互 SID 序列预测下一个商品的 SID。

**Prompt 格式**：
```
### Instruction:
Based on the user's historical interactions, predict the next item's SID.

### Input:
History: <a_34><b_215><c_140>, <a_11><b_88><c_201>, <a_5><b_42><c_99>

### Response:
<a_77><b_156><c_33>
```

### 2.2 辅助任务 1：SidItemFeatDataset

**目标**：建立 SID 与商品标题之间的双向映射。

**双向 Prompt**：

方向 1（SID → Title）：
```
### Instruction:
Given the item's SID, predict its title.

### Input:
SID: <a_77><b_156><c_33>

### Response:
Apple iPhone 14 Pro Max Case
```

方向 2（Title → SID）：
```
### Instruction:
Given the item's title, predict its SID.

### Input:
Title: Apple iPhone 14 Pro Max Case

### Response:
<a_77><b_156><c_33>
```

### 2.3 辅助任务 2：FusionSeqRecDataset

**目标**：融合序列推荐与语义理解，从历史 SID 预测目标商品标题。

**Prompt 格式**：
```
### Instruction:
Based on the user's historical interactions, predict the next item's title.

### Input:
History: <a_34><b_215><c_140>, <a_11><b_88><c_201>

### Response:
Samsung Galaxy S24 Screen Protector
```

### 2.4 多任务设计的意义

| 任务 | 作用 |
|------|------|
| 主任务 | 核心推荐能力：序列 → SID |
| 辅助 1 | SID-语义对齐：让模型理解 SID 的语义含义 |
| 辅助 2 | 跨模态融合：增强模型对商品语义的理解 |

三个任务协同训练，使模型不仅学会 SID 之间的序列模式，还理解 SID 背后的语义信息。

## 三、训练超参数

| 参数 | 值 | 说明 |
|------|-----|------|
| batch_size | 128 | 等效批大小 |
| micro_batch_size | 8 | 单卡实际批大小 |
| gradient_accumulation | 16 | 梯度累积步数 |
| learning_rate | 3e-4 | 学习率 |
| warmup_steps | 500 | 预热步数 |
| num_epochs | 3 | 训练轮数 |
| cutoff_len | 512 | 最大序列长度 |
| 精度 | bf16 | 混合精度训练 |
| gradient_checkpointing | True | 节省显存 |
| early_stopping | patience=3 | 早停策略 |

## 五、训练流程

```
1. 加载基座模型 Qwen2.5-0.5B-Instruct
2. 从 index.json 提取 SID token，扩展 tokenizer 词表
3. resize_token_embeddings 扩展模型 embedding 层
4. 构建三任务 ConcatDataset
5. 配置 Trainer（bf16, gradient_checkpointing, early_stopping）
6. 训练，每 epoch 在 valid 集上评估 loss
7. 保存 final_checkpoint
8. 记录训练日志到 train_log.csv
9. （可选）上传日志到 wandb
```

## 六、断点续训

训练中断后，重新运行相同命令即可自动从最新 checkpoint 续训：

```bash
python phase5_train.py --config train/configs/M_N_50K.json
```

脚本会自动检测 `output_dir` 内的 `checkpoint-*` 目录，从最新的 checkpoint 恢复。

## 八、GRPO 强化学习训练（Phase 6）

在 SFT 模型基础上，使用 GRPO（Group Relative Policy Optimization）强化学习进一步优化推荐性能。核心思想：对同一 prompt 生成一组候选 SID，通过组内相对奖励计算 advantage，鼓励高 reward 的生成。

在 M_N_5K 上仅用 1000 样本 + 1 epoch 即实现 HR@10 最高 +8.1% 的提升。
