## 🎯 核心发现总结

### 1. 幂律 Scaling Law（最重要发现）

```
HR@10(N) = 3.30 × N^(-0.59) + 0.021
R² = 0.995
```

**意义**：生成式推荐的性能随候选集大小呈幂律衰减，N 每增大 10 倍，HR@10 降至原来的 0.26 倍。

### 2. 性能衰减数据

| N | HR@10 (Beam=50) | 相对 5K 衰减 |
|---|-----------------|-------------|
| 5K | 0.0441 | 基准 |
| 20K | 0.0318 | ↓ 27.9% |
| 50K | 0.0260 | ↓ 41.1% |
| 100K | 0.0266 | ↓ 39.7% |

### 3. Beam 收益边际递减

- Beam 20→50: +4%
- Beam 50→100: +2%
- **推荐 Beam=50**（性价比最优）

### 4. 业务阈值

- **HR@10 ≥ 0.05**: N ≤ 3K
- **HR@10 ≥ 0.03**: N ≤ 18K

## 🔬 技术栈总结

### 核心技术

| 技术 | 具体实现 |
|------|---------|
| **基座模型** | Qwen2.5-0.5B-Instruct（24 层，896 hidden） |
| **嵌入模型** | Qwen3-Embedding-0.6B（1024 维） |
| **离散化** | FAISS Residual Quantizer（3 层，codebook=256） |
| **约束解码** | Prefix Trie + ConstrainedLogitsProcessor |
| **训练方法** | 多任务 SFT（主任务 + 2 辅助任务） |
| **评估指标** | HR@10 / NDCG@10 |

### 数据处理技术

| 技术 | 说明 |
|------|------|
| **5-core 过滤** | 迭代过滤至收敛 |
| **分层抽样** | Log-binning + 按比例抽样 |
| **嵌套子集** | S1⊂S2⊂S2.5⊂S3 |
| **Leave-Last-Out** | 时间序列划分 |
| **C=10 限流** | 防止热门商品主导 |
