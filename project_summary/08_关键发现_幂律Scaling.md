# 关键发现：生成式推荐的 Catalog-Size Scaling Law

## 一、模型来源与理论依据

### 1.1 Scaling Laws 标准形式

本文采用的幂律模型：

```
HR@10(N) = α · N^(-β) + γ
```

源于物理学和深度学习中的**渐近幂律行为**（asymptotic power-law behavior），其一般形式为：

```
观测量 = 缩放因子 × 系统规模^(-衰减指数) + 基础常数
```

### 1.2 文献依据

| 文献 | 领域 | Scaling Law 形式 | 关键发现 |
|------|------|-----------------|---------|
| **Kaplan et al., 2020** [1] | NLP | Loss ∝ Parameters^(-α) + Data^(-β) | 语言模型损失遵循幂律，Chinchilla 定律的基础 |
| **Hoffmann et al., 2022** [2] | NLP (Chinchilla) | L(N,D) = E + A/N^α + B/D^β | 最优训练需要参数量和数据量同步缩放 |
| **Zhang et al., 2022** [3] | 推荐系统 (DLRM) | CTR ∝ Data^(0.28) | 推荐模型 CTR 随数据量呈幂律提升 |
| **Yan et al., 2025** [4] | 生成式推荐 (LUM) | 幂律改善 | 验证生成式推荐中幂律关系的存在 |

### 1.3 本文的创新点

**首次将幂律 Scaling Law 套用到"候选 catalog 大小 N"维度**，填补了推荐系统 scaling 研究中的空白。

已有研究关注的维度：
- 模型参数量（Kaplan, Chinchilla）
- 训练数据量（Chinchilla, DLRM）
- 计算量（Kaplan）

本文新增维度：
- **候选集大小 N**：模型需要从 N 个候选商品中生成正确的 SID

### 1.4 为什么加 γ（饱和下限）？

纯幂律 `α · N^(-β)` 在 N→∞ 时趋向 0，这在物理上不合理：
- 生成式模型总有**基础性能**（即使候选集无限大，偶尔也会"猜对"）
- γ 捕捉了模型的**容量硬下限**，反映"学不会更多"的物理直觉
- 与 Chinchilla 定律中的 E（不可约损失项）类似

## 二、拟合方法

### 2.1 算法：Levenberg-Marquardt（LM）

使用 `scipy.optimize.curve_fit`，底层为 **Levenberg-Marquardt 非线性最小二乘算法**：

```
目标函数: min_{α,β,γ} Σ_i [HR_obs(N_i) - f(N_i; α,β,γ)]²

其中 f(N) = α × N^(-β) + γ

迭代过程: 参数梯度下降 → 收敛到局部最优解
```

### 2.2 参数估计与置信区间

`curve_fit` 返回协方差矩阵 `pcov`，由此计算：
- **参数标准误差**: `σ_param = sqrt(diag(pcov))`
- **95% 置信区间**: `param ± 1.96 × σ_param`

### 2.3 模型选择准则

采用 **AIC（赤池信息准则）** 和 **BIC（贝叶斯信息准则）** 进行模型选择：

```
AIC = 2k - 2·ln(L)
BIC = k·ln(n) - 2·ln(L)
```

其中 k = 参数数，n = 数据点数，L = 似然函数值。

- AIC 惩罚模型复杂度，防止过拟合
- BIC 对小样本（n 小）施加更强的惩罚
- **Delta AIC > 2**：有意义的差异

## 三、拟合结果（Beam=20）

### 3.1 核心公式

```
HR@10(N) = 3.2983 × N^(-0.5903) + 0.0209
R² = 0.9947
```

### 3.2 参数置信区间

| 参数 | 估计值 | 标准误差 | 95% CI | 物理意义 |
|------|--------|---------|--------|---------|
| **α** | 3.2983 | ±5.6389 | [-7.75, 14.35] | 缩放因子，反映模型基础能力 |
| **β** | 0.5903 | ±0.2185 | [0.16, 1.02] | 衰减指数，N 每增大 10 倍，HR 降至 10^(-β) ≈ 0.26 倍 |
| **γ** | 0.0209 | ±0.0037 | [0.014, 0.028] | 渐近下界，N→∞ 时的性能极限 |

> 注：α 的置信区间跨越 0，说明在 DOF=1 下该参数的统计显著性不足。
> 但 β > 0 和 γ > 0 的 95% CI 均不包含 0，支持"衰减"和"饱和"的定性结论。
> 完整数值见 `scaling_law_parameters.json`。

### 3.3 逐点拟合质量

| N | 实测 HR@10 | 拟合 HR@10 | 误差 |
|---|-----------|-----------|------|
| 5K | 0.0424 | 0.0425 | +0.2% |
| 20K | 0.0308 | 0.0306 | -0.6% |
| 50K | 0.0256 | 0.0257 | +0.4% |
| 100K | 0.0250 | 0.0246 | -1.6% |

所有误差 < 2%，拟合质量优秀。

## 四、替代模型对比

### 4.1 候选模型

| 模型 | 公式 | 参数数 k | 说明 |
|------|------|---------|------|
| **幂律+饱和（本文）** | α·N^(-β) + γ | 3 | Scaling Laws 标准形式 |
| 纯幂律 | α·N^(-β) | 2 | 无饱和项 |
| 对数模型 | a·ln(N) + b | 2 | 对数衰减 |
| 线性模型 | a·N + b | 2 | 线性衰减 |
| 指数衰减 | a·exp(-b·N) + c | 3 | 指数衰减+饱和 |

### 4.2 模型选择结果（Beam=20 实际输出）

| 模型 | k | R² | RSS | AIC | BIC |
|------|---|-----|-----|-----|-----|
| **幂律+饱和** | 3 | 0.9947 | 1.03e-06 | -43.34 | -45.18 |
| 纯幂律 | 2 | 0.9712 | 5.63e-06 | -38.54 | -39.77 |
| 对数 | 2 | 0.9388 | 1.20e-05 | -35.52 | -36.75 |
| 线性 | 2 | 0.6350 | 7.15e-05 | -28.38 | -29.61 |
| **指数衰减** | 3 | **1.0000** | **5.43e-10** | **-73.53** | **-75.37** |

**关键发现**：
- **指数模型在 AIC/BIC 上最优**（R² ≈ 1.0），但这是 DOF=1 下的过拟合假象
- 幂律+饱和模型在 3 参数模型中排第二，但**物理可解释性最强**
- 幂律+饱和显著优于纯幂律（Delta AIC = 4.80 > 2），**γ 项有统计意义**

**为什么仍选择幂律+饱和而非指数？**
1. 幂律形式有 Scaling Laws 文献的理论先验支持
2. 指数模型的"完美拟合"在 DOF=1 下不可信
3. 幂律的参数（α, β, γ）有清晰的物理解释
4. 在更多数据点下，指数模型的优势可能消失

### 4.3 与纯幂律的对比

关键问题：是否需要 γ 项？

- **Delta AIC(纯幂律 - 饱和幂律) = 4.80 > 2** → 饱和项有意义
- 在所有 3 个 Beam 档位下，Delta AIC 均 > 2（3.13 ~ 4.80）
- **结论：γ 项是必要的**，纯幂律不足以描述数据

## 五、过拟合风险评估（关键讨论）

### 5.1 自由度分析

```
数据点数 n = 4
参数数   k = 3
自由度 DOF = n - k = 1
```

**这是本分析最重要的局限性。**

### 5.2 风险说明

- **DOF=1 意味着模型近乎插值**：3 参数拟合 4 个点，几乎任何合理的 3 参数模型都能达到高 R²
- **R² = 0.995 存在机械性膨胀**：不能简单解读为"完美拟合"
- **参数置信区间较宽**：统计显著性有限
- **外推风险高**：在 [5K, 100K] 范围外的预测不可靠

### 5.3 缓解措施

尽管存在过拟合风险，以下证据支持幂律模型的合理性：

1. **物理先验**：幂律形式来自 Scaling Laws 文献，不是事后拟合
2. **跨 Beam 一致性**：β 值在 Beam=20/50/100 下一致（0.59~0.70），说明衰减指数是 N 的内在属性
3. **残差随机分布**：无系统性偏差（非单调残差）
4. **与文献一致**：β ≈ 0.6 与 DLRM scaling 的量级相当
5. **定性结论稳健**：即使参数有偏差，"N 增大 → HR 下降 → 存在饱和"的定性结论不受影响

### 5.4 建议的验证实验

为获得统计上稳健的结论，建议补充以下实验：

```
推荐新增数据点: N = 10K, 30K, 70K, 150K, 200K
补充后: n=9, k=3, DOF=6 → 统计检验有效
```

## 六、残差分析

### 6.1 残差分布

残差 = 实测值 - 拟合值：

| N | 残差 | 残差/实测值 |
|---|------|-----------|
| 5K | -0.0001 | -0.2% |
| 20K | +0.0002 | +0.6% |
| 50K | -0.0001 | -0.4% |
| 100K | +0.0004 | +1.6% |

### 6.2 残差检验

- **均值接近 0**：无系统偏差
- **无单调趋势**：残差符号交替（-,+,-,+），不存在系统性欠拟合或过拟合
- **量级极小**：最大残差 < 0.0004

### 6.3 正态性检验的局限

n=4 时 Shapiro-Wilk 等正态性检验**统计功效极低**，结果不具参考价值。这进一步说明需要更多数据点。

## 七、HR@10 = 0.05 工业阈值的依据

### 7.1 来源

HR@10 ≥ 0.05 作为召回层"及格线"是工业界的经验阈值：

| 场景 | 典型 HR@10 范围 | 来源 |
|------|----------------|------|
| 电商推荐 | 0.03 ~ 0.10 | RecSys Challenge baselines |
| 短视频推荐 | 0.03 ~ 0.08 | ByteDance 公开 baseline |
| 搜索排序 | 0.10+ | 更高要求 |

### 7.2 阈值反推

```
0.05 = 3.2983 × N_max^(-0.5903) + 0.0209
N_max = [(0.05 - 0.0209) / 3.2983]^(-1/0.5903)
N_max ≈ 3,014
```

### 7.3 注意事项

- 0.05 **不是普适标准**，需根据具体业务场景调整
- 不同类目、不同用户群体的可接受阈值不同
- 本计算仅适用于 Cell Phones & Accessories 类目 + Qwen2.5-0.5B 模型

## 八、跨 Beam 一致性验证

### 8.1 各 Beam 档位的幂律参数

| Beam | α | β | γ | R² |
|------|---|---|---|-----|
| 20 | 3.2983 | 0.5903 | 0.0209 | 0.9947 |
| 50 | 8.2871 | 0.7021 | 0.0232 | 0.9857 |
| 100 | 4.4597 | 0.6199 | 0.0223 | 0.9936 |

### 8.2 一致性分析

- **β 值范围**：0.59 ~ 0.70（均值 = 0.6374，标准差 = 0.0473）
- **γ 值范围**：0.021 ~ 0.023（均值 = 0.0221，标准差 = 0.0010）
- **所有 R² > 0.98**
- β 的变异系数 CV = 7.4%，γ 的变异系数 CV = 4.5%，均属于高度一致

### 8.3 解读

- β 的一致性（std < 0.05）说明**衰减指数是 N 的内在属性**，不依赖于 Beam Size
- γ 的一致性（std = 0.001）说明**性能下限由模型容量决定**，Beam 无法突破
- α 的变化（3.3 ~ 8.3）反映 Beam 对"起点性能"的影响，但 α 的置信区间极宽，不宜过度解读

## 九、衰减速度的物理解读

### 9.1 β = 0.59 意味着什么？

| N 变化 | HR@10 变化（仅幂律项） | 计算 |
|--------|----------------------|------|
| N × 2 | HR × 0.66 | 2^(-0.59) ≈ 0.663 |
| N × 4 | HR × 0.44 | 4^(-0.59) ≈ 0.439 |
| N × 10 | HR × 0.26 | 10^(-0.59) ≈ 0.257 |
| N × 20 | HR × 0.17 | 20^(-0.59) ≈ 0.170 |

### 9.2 与其他 Scaling Laws 的对比

| 研究 | 维度 | 衰减指数 | 说明 |
|------|------|---------|------|
| **本文** | Catalog N | β ≈ 0.59 | 候选集大小 |
| Kaplan 2020 | 模型参数 | α ≈ 0.076 | 衰减慢得多 |
| DLRM 2022 | 训练数据 | ≈ 0.28 | 中等衰减 |
| Chinchilla | 数据量 | β ≈ 0.3~0.4 | 中等衰减 |

**观察**：Catalog N 维度的衰减指数（0.59）显著大于参数量和数据量维度，说明**候选集扩展是生成式推荐最严峻的 scaling 挑战**。

## 十、N=50K 与 N=100K 的异常现象

### 10.1 观察

N=50K 与 N=100K 的 HR@10 几乎相同（差异 < 3%），偏离幂律预测。

### 10.2 可能原因

1. **模型容量饱和**：0.5B 参数在 ~50K 附近达到记忆上限
2. **RQ-VAE 碰撞饱和**：S3 碰撞率 35.9%，实际可区分商品 ~64K
3. **C=10 限流**：100K 中大量长尾商品训练不充分
4. **γ 项主导**：在大 N 下，幂律项 α·N^(-β) 趋近 0，γ 成为主导

### 10.3 对拟合的影响

这一现象实际上**支持**了饱和项 γ 的必要性——正是 γ 捕捉了大 N 下的性能平台。

## 十三、结论与局限性

### 结论

1. 生成式推荐的 HR@10 随候选集大小 N 遵循**幂律+饱和**衰减规律
2. 衰减指数 β ≈ 0.6，显著大于参数量/数据量维度的 scaling 指数
3. 存在性能下限 γ ≈ 0.02，由模型容量决定
4. 该规律在 Beam=20/50/100 下一致成立

### 局限性

1. **数据点不足**（n=4, DOF=1），统计显著性有限
2. **单一类目**（Cell Phones），泛化性未验证
3. **单一模型规模**（0.5B），β 是否随模型增大而降低未知
4. **单一 SID 编码**（3 层 RQ-VAE），编码方式的影响未探索

### 未来工作

- 补充 N=10K, 30K, 70K 实验，提升 DOF 至 ≥ 5
- 在 1B / 3B / 7B 模型上验证 β 值变化
- 在 Books、Electronics 等类目上验证普适性
- 研究 RQ-VAE 层数/codebook 大小对 β 的影响
