## 二、各 Python 文件职责一览

| 文件 | 阶段 | 职责 |
|------|------|------|
| `phase1_data_loading.py` | Phase 1 | 从 HuggingFace 加载 Amazon Reviews 2023；提取 user_id/parent_asin/timestamp；5-core 过滤；统计与 popularity 可视化；加载 item 元数据；保存 Parquet |
| `phase2_subset_construction.py` | Phase 2 | 基于 popularity 做 log-binning；分层抽样构造 S3⊃S2.5⊃S2⊃S1；检查嵌套关系；绘制 popularity 分布 |
| `phase2_5_validation.py` | Phase 2.5 | 校验元数据覆盖、时间戳格式、重复交互；分析用户序列长度；演示 leave-one-out 切分 |
| `phase3_sequence_construction.py` | Phase 3 | 为 S1–S3 构造序列：过滤子集、按用户分组排序、截断到 20、leave-last-out、滑动窗口训练样本、C=10 限流 |
| `phase4_sid_generation.py` | Phase 4 | 用 Qwen3-Embedding-0.6B 生成 item 嵌入；在 S3 上训练 FAISS RQ-VAE；为各子集生成 SID 映射与 prefix trie |
| `phase5_train.py` | Phase 5 | 多任务 SFT：SidSFTDataset + SidItemFeatDataset + FusionSeqRecDataset；扩展 tokenizer 词表；训练 Qwen2.5-0.5B-Instruct |
| `eval/run_eval.py` | Phase 6 | 单模型评估：ConstrainedLogitsProcessor 约束 SID 解码；计算 HR@K / NDCG@K |
| `eval/run_all_eval.py` | Phase 6 | 批量评估多模型、多 beam；一次加载模型，多次评估 |
| `eval/build_trie.py` | Phase 6 | 从 index.json 或 prefix_trie.json 构建 SID Prefix Trie 用于约束解码 |
| `analysis/plot_results.py` | Phase 7 | 读取 results_summary.json；绘制 HR@10 vs N 曲线、Beam×N 热力图；输出文本结论 |
| `upload_wandb.py` | 辅助 | 将本地 train_log.csv 上传到 wandb 项目 |

### 训练配置示例（`train/configs/M_N_100K.json`）

```json
{
  "base_model": "Qwen/Qwen2.5-0.5B-Instruct",
  "train_file": "data/sequences/S3/train.csv",
  "eval_file": "data/sequences/S3/valid.csv",
  "sid_index_path": "data/sequences/S3/S3.index.json",
  "item_meta_path": "data/sequences/S3/S3.item.json",
  "output_dir": "train/runs/M_N_100K",
  "batch_size": 128,
  "micro_batch_size": 8,
  "num_epochs": 3,
  "learning_rate": 3e-4,
  "cutoff_len": 512,
  "warmup_steps": 500
}
```

- `models`: M_N_5K / M_N_20K / M_N_50K / M_N_100K

- `topk`: 10
