# Cell Phones & Accessories 生成式序列推荐系统 — 项目总览

## 一、研究背景

本项目是一个**生成式序列推荐（Generative Sequential Recommendation）**实验管线，参照 **MiniOneRec** 框架设计。核心思想是：将传统推荐系统中的"检索 + 排序"范式转变为**语言模型直接生成目标商品 ID** 的范式。

具体而言，项目为每个商品分配一个**语义 ID（Semantic ID, SID）**，然后利用大语言模型（LLM）学习用户的历史交互序列，直接生成下一个推荐商品的 SID。数据领域为 Amazon Reviews 2023 的 **Cell Phones and Accessories** 类目。

**核心研究问题**：当候选商品集大小 N 从 5K 增大到 100K 时，生成式推荐的 HR@10 如何衰减？不同 Beam Size 对性能的影响如何？

---

## 二、整体流水线（7 个阶段）

```
Phase 1: 数据载入与清洗          → phase1_data_loading.py
Phase 2: 嵌套候选集构造（分层抽样）→ phase2_subset_construction.py
Phase 2.5: 数据质量验证           → phase2_5_validation.py
Phase 3: 序列构造 + 数据划分      → phase3_sequence_construction.py
Phase 4: 语义嵌入 + RQ-VAE → SID → phase4_sid_generation.py
Phase 5: 多任务 SFT 训练          → phase5_train.py
Phase 6: 约束解码评估             → eval/run_eval.py, eval/run_all_eval.py
Phase 7: 结果可视化与分析         → analysis/plot_results.py
```

## 三、技术亮点

| 技术点 | 说明 |
|--------|------|
| **语义 ID（SID）** | 通过 RQ-VAE 将商品嵌入离散化为 3 层 token，兼顾语义相似性和可生成性 |
| **嵌套子集设计** | S1⊂S2⊂S2.5⊂S3，控制变量研究 N 对性能的影响 |
| **分层抽样** | Log-binning + 按比例抽样，保证各子集 popularity 分布一致 |
| **C=10 限流** | 防止热门商品主导训练集，提升长尾商品覆盖 |
| **多任务 SFT** | 主任务 + 2 个辅助任务，增强 SID 与语义的对齐 |
| **Prefix Trie 约束解码** | 保证生成的 SID 一定合法，提升推理准确性 |
| **端到端管线** | 从原始数据到评估可视化，7 个阶段完整自动化 |
